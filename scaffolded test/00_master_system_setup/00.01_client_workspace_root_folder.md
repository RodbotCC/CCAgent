# 00.01 Client Workspace Root Folder

## Deliverable

Create the top-level workspace location where every client or lead folder will live.

## Purpose

This root folder is the system's operational home for all lead-specific work. Every harvested lead eventually gets a dedicated folder underneath this root, and all master-level ledgers, registries, indexes, logs, skills, automations, dashboards, and shared configuration files live here or point back here.

## Proposed Root

```text
scaffolded test/
```

## Root Folder Rules

1. All per-lead/client folders should be created under `scaffolded test/clients/`.
2. Master-level tracking artifacts should be stored under `scaffolded test/master_ledgers/`, `scaffolded test/registries/`, `scaffolded test/indexes/`, and `scaffolded test/logs/`.
3. Stage-specific build documents should be stored in numbered stage folders matching the blueprint.
4. Shared templates should live in `scaffolded test/templates/`.
5. Nothing lead-specific should be stored only at the root. The root is for navigation, orchestration, and shared system state.

GitHub does not track empty folders directly, so each future folder will become real in the repository once it contains at least one file. This first deliverable establishes the root namespace and folder convention for the rest of the semi-automated scaffold test.
