# 00.03 Master Smart View Registry

## Deliverable

Create the master registry that stores which Close smart views are active sources of leads.

## Purpose

The master smart view registry tells the system which Close smart views are approved lead sources. It prevents the agent from guessing which Close view to use, keeps intake repeatable, and gives Andre a clean way to change or pause lead sources without rewriting the rest of the pipeline.

## Recommended Location

```text
scaffolded test/registries/master_smart_view_registry.md
```

## Registry Responsibilities

This registry should track:

1. The human-readable smart view label
2. The Close smart view URL
3. The Close smart view ID, when available
4. Whether the smart view is active, paused, deprecated, or test-only
5. What kind of leads the view contains
6. Who approved the view for intake
7. When it was last verified
8. Which harvest runs used it
9. Any notes about filtering, priority, or expected lead quality

## Suggested Columns

| Column | Purpose |
|---|---|
| Smart View Label | Human-readable Close view name |
| Smart View ID | Unique Close smart view identifier |
| Smart View URL | Direct URL to the Close smart view |
| Status | active, paused, deprecated, test_only |
| Lead Type | Wedding, corporate, tasting, reactivation, whale, etc. |
| Priority | high, normal, low |
| Approved By | Person who confirmed this view should be used |
| Last Verified | Date/time the view was last confirmed valid |
| Last Harvest Run | Most recent run that used this view |
| Expected Use | What the system should do with this view |
| Notes | Operational notes or caveats |

## Status Vocabulary

```text
active
paused
deprecated
test_only
unknown
```

## Starter Template

```markdown
# Master Smart View Registry

| Smart View Label | Smart View ID | Smart View URL | Status | Lead Type | Priority | Approved By | Last Verified | Last Harvest Run | Expected Use | Notes |
|---|---|---|---|---|---|---|---|---|---|---|
| _pending_ | _pending_ | _pending_ | unknown | _pending_ | normal | _pending_ | _pending_ | _none_ | _pending_ | _none_ |
```

## Registry Rules

1. Only smart views listed as `active` should be used for normal lead harvest.
2. `test_only` views may be used for sandbox runs but should not feed production automations.
3. `paused` views should remain visible but should not be harvested until reactivated.
4. `deprecated` views should remain for history but should not be used.
5. Every harvest run should record which smart view it used.
6. If a smart view URL changes, the old URL should be retained in notes or history before replacement.
