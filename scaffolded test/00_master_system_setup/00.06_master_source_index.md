# 00.06 Master Source Index

## Deliverable

Create the master source index that tracks where every source artifact came from.

## Purpose

The master source index is the system-wide citation and provenance layer. It records every source artifact used during intake, comms retrieval, enrichment, research, synthesis, planning, automation, and dashboard work.

The goal is to make every downstream profile, plan, decision, asset, and automation traceable back to its original source.

## Recommended Location

```text
scaffolded test/indexes/master_source_index.md
```

## Source Index Responsibilities

This index should track:

1. Close smart view URLs
2. Individual Close lead URLs
3. Raw Close communication captures
4. SMS, email, call, task, note, and activity timeline source references
5. Public web sources found during people search
6. Public web sources found during deep dive research
7. Public social/profile/media sources
8. Business websites, registry pages, review pages, press pages, and public filings
9. Screenshots and captured media assets
10. Source capture timestamps
11. Source confidence and source type
12. Which lead or system artifact used the source
13. Whether a source is active, archived, unavailable, duplicate, or out of bounds

## Suggested Columns

| Column | Purpose |
|---|---|
| Source ID | Stable identifier for the source row |
| Source Type | Close, web, social, media, business, asset, internal, generated, other |
| Source Label | Human-readable description of the source |
| Source URL | Direct link or canonical source location |
| Source Origin | Where/how the source was discovered |
| Lead Registry ID | Related lead, if applicable |
| Lead Name | Human-readable lead name, if applicable |
| Related Artifact | File or output that used this source |
| Capture Timestamp | When the source was captured or referenced |
| Last Verified | Most recent verification timestamp |
| Status | active, archived, unavailable, duplicate, out_of_bounds, unknown |
| Confidence | high, medium, low, unknown |
| Notes | Context, limitations, or caveats |

## Source ID Convention

Use simple ascending IDs:

```text
source-000001
source-000002
source-000003
```

Lead-specific source indexes can use the same source IDs or add lead-local references, but the master source index should preserve the global source record.

## Indexing Rules

1. Any external source used to create a profile, plan, logic file, or outbound strategy must be indexed.
2. Close source URLs should be indexed even if their content is later copied into a local markdown file.
3. Screenshots, profile images, logos, videos, and media references should be indexed both here and in the asset index.
4. Sources that are searched but not useful do not all need full entries, but important searched-and-not-found categories should be summarized in the relevant checklist or search log.
5. Out-of-bounds sources should be labeled rather than silently deleted when they explain why something was not used.
6. Generated files should be indexed when they become sources of truth for later stages.
7. The index should prefer canonical URLs over temporary or redirected URLs.
