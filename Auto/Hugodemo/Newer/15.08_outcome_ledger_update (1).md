# 15.08 — Outcome Ledger Update (process note)

Whenever an outcome file gets filled, the corresponding row in
`master_ledger.csv` updates `17.30 current_live_state`:

| outcome | live_state |
|---|---|
| 15.01 close | close |
| 15.02 follow-up | active |
| 15.03 hold | hold |
| 15.04 block | block |
| 15.05 nurture | nurture |
| 15.06 referral | (additive — doesn't change live_state) |
| 15.07 post-close handoff | complete |

The Liaison Agent does NOT change `live_state` autonomously — every
state change is operator-acknowledged. The agent only proposes the
state change with a draft outcome file.
